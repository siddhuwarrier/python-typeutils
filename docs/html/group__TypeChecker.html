Decorator for type checking.

This method performs type checking for Python function arguments. This code is modified from Per Vognsen's ActiveState recipe 454322 http://code.activestate.com/recipes/454322/ Vognsen's code has been cleaned up and modified in order to support checking multiple kwargs. It also performs checks as to whether the kwarg provided is valid.

Example: 

from sidutils.TypeChecker import require

@require(validKwargs = ['z'], x = int, y = (int, float), z = str) #x can only be int, y can be int or float.
def foo(x, y, **kwargs):
        pass
if __name__ == "__main__":
    foo(3,5.6) #will work
    foo(3, 5) #will work
    foo(x = 3, y = 5) #will work
    try:
        foo(x = "string", y = 2) #TypeError
    except TypeError as typeError:
        print typeError.args[0]
    foo(x = 23, y = 5, z = "ss") #will work
    foo(23, 5, z = "superman") #will work
    try:
        foo(5,5, m = "xd") #NameError
    except NameError as nameError:
        print nameError.args[0]
Parameters:
validKwargs (list): The list of valid keyword arguments for the method/function being decorated.
kwargsCompulsory (boolean): Set to indicate whether the function's keyword arguments (in validKwargs) should be compulsory or not. Default: False.
typeMap (dictionary of tuples/strings): The typemap is a dictionary of the argument names for which we want to perform type checking. Each key specifies the acceptable type(s) for the argument specified in the key. If there is more than one type, the key contains a tuple; if not, it may contain either a tuple or a string.
Exceptions:
NameError When argument invalid, or required argument not found.
TypeError When argument type does not match argument type expected.
